Overview

Javascript - being one of the most popular programming languages, has a lot of advantages but it comes with a few drawbacks as well. Among its merits; it is fairly easy to learn and one can write javascript code with minimal programming background.

The problems arises when the programmer starts to nibbling areas such as asynchronous code - especially in context of Node.js. Concurrency becomes a big issue since there is no way to tell when a certain process finishes. This problem has been addressed by writing nested functions but very quickly, the development community recognized that this might not the optimal solution. Enter futures and promises - theoretically an elegant solution but in practice it comes with drawbacks; above all the concept is hard to digest for many.

What's even more problematic is that developers have trouble understand, follow or debug large codebase written by colleagues or even code by him-/herself a few months later, if asynchronous programming is involved - which is very common.
